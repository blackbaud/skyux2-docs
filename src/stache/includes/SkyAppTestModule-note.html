Note that in the sample unit tests, the configureTestingModule function imports SkyAppTestModule for testing convenience. This module brings in your all of your SPA's components and modules. By default, SPAs import the entire SKY UX library, so SkyAppTestModule imports all components and modules into your tests. This can negatively impact test performance for applications that only use a subset of SKY UX components, but we still recommend that you import SkyAppTestModule into your TestBed. To optimize performance when tests don't require access to all SKY UX components and modules, use the skyuxModules configuration property to specify the SKY UX modules to import into your application. This option provides a balance between performance and the simplicity that SKY UX Builder provides.
