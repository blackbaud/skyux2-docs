--template or -t — Specifies the base template to use to create items through the CLI. Supported values include:
  • -t addin — References the SKY UX Add-in template to create SKY Add-ins that allow developers to build features that integrate with Blackbaud solutions.
  • -t barebones — References a pared-down version of the SKY UX Add-in template to create SKY Add-ins that allow developers to build features that integrate with Blackbaud solutions. This template eliminates folders, modules, and references to packages that are not required to build add-ins.
  • -t library — References the SKY UX Library template to create SKY UX libraries that allow developers to build and publish reusable front-end libraries for other SPAs to consume.
  • -t stache — References the Stache template to create public Stache sites for technical documentation.
  • -t <ANY-GIT-URL> — Allows for complete flexibility by cloning any valid Git URL.
The template's repo requires a versioned branch that the SKY UX CLI controls. It follows the [SKY UX CLI major version].x.x format. As of SKY UX 4, the 4.x.x branch must exist.
