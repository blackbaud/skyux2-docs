--browser or -b â€” Specifies the browser to use. This option is useful for cross-browser manual testing. The available values are platform-dependent, and they correspond to the names of the browser executables on each platform. For example, the value for Chrome is google chrome on macOS, google-chrome on Linux, and chrome on Windows. Microsoft Edge is an exception that is not platform-dependent, and its value is edge on all platforms. You can also pass in the browser's full path. SKY UX passes the value to the open library as the app property, so for more information, see the open library's README file. You can specify multiple --browser arguments. The first argument specifies the browser to use, while subsequent arguments are passed to the browser. This is useful for serving the application in incognito/private mode. For example, to run the application in Chrome on Linux and use inincognito mode, enter skyux serve --browser='google-chrome' --browser='--incognito'.
