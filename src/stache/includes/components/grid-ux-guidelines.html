Usage

The grid provides a layout for tabular data.

When to use grids

Use grids to display tabular data when:

  • A large amount of data needs to be visible.
  • Users need to visually scan columns for certain values or outliers.
  • Users need to compare values between rows.
When not to use grids
  • Do not use grids when you need to display visual content such as graphs or images. Use repeaters or card views instead.
  • Do not use grids when the content calls for a more complex layout. If grids require a lot of templated columns, try repeaters instead.
  • Do not use grids when you need to display many fields and users are likely to view the grids in a small viewport such as a phone. Try repeaters instead.
Anatomy
  1. List count (optional)
  2. Toolbar (optional)
  3. Multiselect toolbar (optional)
  4. Header row
  5. Row selection checkbox (optional)
  6. Context menu button (optional)
  7. Data
  8. Alternate row striping
  9. Row divider
  10. Selected row background
  11. Pagination (optional)
  12. Summary action bar (optional)
Options
Context menu

Use context menus in the first column when users need to access multiple actions in the rows.
Multiselect

Use multiselect when:

  • Users need to act on more than one row at a time.
  • The list is presented as a full page, a full-page modal, or a full page within a flyout.

Don't use multiselect when:

  • The list is presented in a tile, page section, or other container that might flow off the screen.

To allows users to select or clear all items and to view only the selected items, include the multiselect toolbar.

When users select rows, the summary action bar should fly in from the bottom of the page. Place actions that can be taken on the selected items in the summary action bar.
Sorting

If users are likely to sort based on the data in columns, enable sorting so that they can click column headers to toggle between sorting in ascending and descending order for the specified fields in columns. If users need to sort by data in a templated column, use the sort component instead to allow them to choose the data to sort on.
Filtering

If users are likely to work with a subset of items in the grid, enable filtering so that they can focus on those items.
Templated columns

When columns need to represent composite information instead of single values, use templated columns to lay out content as necessary.
Behavior
Fit
Scroll

Setting the fit property to scroll allows the grid to take as much space as necessary by adding a horizontal scrollbar. This is most useful for grids with large numbers of columns.
Width

Setting the fit property to width fixes the width of the grid to that of its container. Columns resize themselves to fit. This is most useful for simple grids in smaller containers such as tiles where the content fits neatly in the allotted space.
Reordering columns

Users can reorder columns by dragging and dropping.
Resizing columns

Users can resize columns by clicking and dragging or by using the keyboard.
Responsiveness

Grids tend to perform poorly in small viewports. The columns in fixed-width grids shrink down to the point of being unreadable, and scrolling grids require a lot of interaction to see all the content. Use repeaters instead if possible.
