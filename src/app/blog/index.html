Engineering update - October 29, 2019

New SKY UX CLI command to manage local development certificate

The intermediate SKY UX certificate that developers need for local development expires on Oct. 31, 2019. skyux serve, skyux e2e, and skyux build –-serve will not function correctly without it.

To address this problem, we re-evaluated our implementation of certificate management and drew inspiration from contemporary tools such as the .NET Core CLI. Our solution introduces a more secure, robust approach where each developer generates a unique trusted certificate.

To continue with local development, SKY UX developers must globally upgrade to the latest version of @skyux-sdk/cli and generate a new local certificate. They must also upgrade all SKY UX SPAs, SKY UX libraries, and Stache sites to the latest version of @skyux-sdk/builder.

To upgrade @skyux-sdk/cli and generate a certificate, run the following commands globally:

  • npm i -g @skyux-sdk/cli@latest
  • skyux certs install

To upgrade @skyux-sdk/builder, run the following command for each SKY UX SPA, SKY UX library, and Stache site:

  • npm i --save-dev --save-exact @skyux-sdk/builder@latest

For more information, including more specific instructions for Firefox and iOS Simulator, see the SKY UX prerequisites.
