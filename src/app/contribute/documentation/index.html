We recommend that you create reference documentation for the Components section of the SKY UX website for the components or features you contribute. Component documentation should explain all features of your components, including all properties and events. In the SKY UX docs repo, you create subfolders for your components in the src/app/components folder and document them in index.html files. To create action buttons for the Components section landing page, you add entries in the demo-components.service.ts file.
