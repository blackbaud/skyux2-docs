The SKY UX framework provides functionality through multiple repositories that target different areas of development. To learn how we organize these packages, see the packages matrix. For information about how to contribute to a particular package, see the CONTRIBUTING.md file in its repo.

Contributing to SKY UX A step-by-step guide to contributing to SKY UX libraries. File an issue

Before you start any work, file an issue. The sooner we can discuss potential code changes, the better. To file a issue with SKY UX, follow these instructions:

  1. Create an issue in the applicable repo.
  2. Delete the default fields for reporting bugs and describe the contribution and potential code changes to SKY UX.
  3. For new components or significant features, you must include design documents and requirements.
  4. Subscribe to the issue to receive notifications.
Discuss with the SKY UX team Once the issue is ready, meet with the SKY UX team to discuss architecture and implementation details, requirements, and timelines. A SKY UX developer will be assigned to the issue; they can help answer any questions you may have. Write the feature code Create a fork of the appropriate repo and push code to a kebab-cased feature branch. You don't need to write tests or documentation yet. Get a preliminary review Tag the SKY UX team in the issue to request a review of the initial implementation, functionality, and styling. Make changes and get additional reviews as needed. For large changes that include multiple components (more than 50 files), please submit changes in several small pull requests instead of one large pull request. We recommend waiting to add tests and documentation until after this review, as the implementation details could change. Add tests All new code should have 100% unit test code coverage. Components should have visual regression tests and accessibility tests as well. Get a final review Get a review of the complete contribution. We'll go over the implementation, testing, functionality, styling, localization, and accessibility. Document your contribution Write or update the documentation for your contribution. For components, file an issue in the SKY UX docs repo and work with a SKY UX docs writer. You can find more information on writing docs here.
