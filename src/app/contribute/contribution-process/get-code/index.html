To contribute to SKY UX, fork the appropriate repo and create a kebab-case branch to commit and push changes.

  1. Fork the repo into your own GitHub repo. For a list of the SKY UX repositories that target different areas of development, see the packages matrix.
  2. Create a branch off of master. Name it after the feature to contribute, and use kebab-case (my-new-feature).
  3. Clone your repo locally and run nvm use to ensure you have the preferred version of Node.js installed. Then run npm install to install all required dependencies. For more information on prerequisites, see SKY UX prerequisites.
  4. For components, run your first round of visual regression tests with skyux e2e. This establishes baseline screenshots of the components in the master branch so that as you make changes and run visual regression tests, your changes are validated against the state of the repo before the changes.
