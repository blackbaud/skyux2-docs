Before you start any work, file an issue. If an issue already exists, comment on it and tag the SKY UX team so that we know you intend to work on it. The sooner we discuss potential code changes, the better.

  1. Create an issue in the package's repo. SKY UX provides functionality through multiple repositories that target different areas of development, so use the packages matrix to find the appropriate repo to file your issue.
  2. Delete the default fields for reporting bugs, and describe the contribution and its potential code changes.
  3. For new components and significant features, include design documents and requirements.
  4. Subscribe to the issue to receive notifications.
