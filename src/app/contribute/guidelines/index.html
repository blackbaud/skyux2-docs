The SKY UX framework provides functionality through multiple repositories that target different areas of development. To learn how we organize these packages, see the packages matrix. For information about how to contribute to a package, see the CONTRIBUTING.md file in its repo.

We are expanding the contribution guidelines on this site, but the guidelines on this page are currently a work in progress. For additional guidance on SKY UX, see the contribution guidelines in the SKY UX repo. Submit code
  1. Commit and push changes to a forked version of the repo.
  2. Submit a pull request.
Report bugs

If you uncover a bug while working with a package, please file an issue in its repo. We use issues to track and assign work.

  1. Create a GitHub issue in the applicable repo.
  2. Fill out the template fields to provide:
    • The expected behavior
    • The actual behavior
    • Steps to reproduce
    • A Plunkr example, if applicable
    • The severity
    • The impact
  3. Subscribe to the issue to receive notifications.
