Report Bugs

If you uncover a bug while working with a package, please file an issue in its repo. We use issues to track and assign work.

  1. Create a GitHub issue in the applicable repo.
  2. Fill out the template fields to provide:
    • The expected behavior
    • The actual behavior
    • Steps to reproduce
    • A Plunkr example, if applicable
    • The severity
    • The impact
  3. Subscribe to the issue to receive notifications.
Recommended tools

Visual Studio Code setup

Like Angular, SKY UX components are written in TypeScript, and the free Visual Studio Code editor is a great IDE to work with TypeScript. It supports code navigation, refactoring tools, and a host of other features that are usually only found in richer IDEs. The SKY UX team uses VS Code and highly recommends the TSLint and EditorConfig extensions to configure it to work best with Angular and SKY UX. If you do not use VS Code, we still recommend these extensions for other popular editors such as Atom, Brackets, and Sublime Text.

  • TSLint validates your TypeScript code against the rules specified by the tslint.json file in the SKY UX repo. TSLint runs during the CI build, so if this extension does not validate your code in VS Code, then the CI build will fail.
  • EditorConfig allows VS Code to recognize the .editorconfig file and automatically enforces many coding conventions that the SKY UX team uses, such as indent size and trailing whitespace rules. If you do not use this extension, then you will find yourself fixing a lot of TSLint errors manually instead of letting VS Code do it for you.
