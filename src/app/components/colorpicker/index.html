 Specifies the type of transparency to use. Specifies an initial color to load in the colorpicker. This property is deprecated. As an alternative, we recommend the formControlName property on reactive forms or ngModel on template-driven forms. See the demo below for examples. Specifies the format to use for the value in a colorpicker when the colorpicker uses a native input element such as a standard text input or a button. This property accepts rgba, hex, or hsla, but we do not recommend using it because users never see or use its value. Instead, if you need to access this format value, see the demo below for an example of the recommended way to access the value. Specifies an array of colors to load in the colorpicker as preset choices. The colorpicker displays the colors in a series of 12 boxes for users to select. Place this attribute on an input element or button element that is wrapped in a sky-colorpicker component to create the element and colorpicker. You must set this attribute to the instance of the sky-colorpicker. This directive uses NgModel to bind data.
