The colorpicker module contains directives to let users select colors. It is a SKY UX-themed replacement for the HTML input element with type="color". The value that users select is driven through the ngModel attribute specified on the input element. Specifies a message stream to toggle the reset button on and off. Indicates whether to display a reset button to let users return to the default color. Fires when users click Apply in the colorpicker to apply a color. Fires when users select a color within the colorpicker.
Both events return an object with the following properties:

Creates the colorpicker element and dropdown. Place this attribute on an input element or button element, and wrap the element in a sky-colorpicker component. This attribute must be set to the instance of the sky-colorpicker component. Indicates whether to display a transparency slider for users to select transparency levels. Specifies the type of transparency to use in the transparency slider. Specifies an initial color to load in the colorpicker. This property is deprecated. As an alternative, we recommend the formControlName property on reactive forms or ngModel on template-driven forms. See the demo for examples. Specifies the format to use for the color when the colorpicker uses a native input element such as a standard text input or a button. This property accepts rgba, hex, or hsla, but we do not recommend using it because users never see or use its value. Instead, if you need to access this format value, see the demo for an example of the recommended way to access the value. Specifies an array of colors to load as preset choices. The colorpicker displays the colors in a series of 12 boxes for users to select. This property is deprecated and does not affect the colorpicker. We recommend against using it.
