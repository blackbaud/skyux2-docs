The date range picker component creates a text input with a dropdown for users to select a date range from a set of well-known options. This includes options that enable users to specify dates to create their own date range. The text input is a composite component of up to two sky-datepicker components. The form model for the date range picker field returns a value with a type of SkyDateRangeCalculation.

Specifies the calculator that determines the dates in the date range. This property returns a SkyDateRangeCalculatorId value. Specifies the last date in the date range. This property returns a Date value. Specifies the first date in the date range. This property returns a Date value.

SkyDateRangeCalculatorId values specify calculator objects that return two Date objects to represent date ranges. The SkyDateRangeCalculatorId values are used to populate the options in the date range picker's dropdown.

SKY UX uses the SkyDateRangeService to create calculators and configures each one with a validate function to confirm that the dates users select are compatible. For example, validate functions ensure that start dates are before end dates. SKY UX also configures calculators to call a getValue function after the validate function and return a range of two Date objects.

Each calculator in the date range picker's dropdown has its own validator that runs when the value changes. Errors are returned in the following format:

{ skyDateRange: { calculatorId: SkyDateRangeCalculatorId, errors: { [nameOfTheError: string]: boolean; } } }
Usage

Use the date range picker when users need to select a range of dates or a single date relative to the current day, such as yesterday.

Behavior
Specific range

When users select "Specific range," from and to date fields appear.
Responsiveness

On smaller viewports, the date fields that appear when users select "Specific range" reflow into a one-column layout.
Layout
Error message

Display validation error messages below the date range picker.
