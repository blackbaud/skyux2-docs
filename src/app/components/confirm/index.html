The confirm service launches simple confirmation dialogs to allow users to confirm actions. The SkyConfirmService provider launches the confirmation dialog. Within the service, you can specify the dialog's message and customize the button text. Opens a confirmation dialog and passes the config parameter of type SkyConfirmConfig to the component's constructor. This method returns an object of type SkyConfirmInstance. Specifies the message to display in a bold font at the top of the dialog. This property accepts string values. Specifies secondary text to display in plain font under the primary message. This property accepts string values. Specifies an array of SkyConfirmButtonConfig objects that overwrite the default configuration for buttons. The number of items in the array must match the number of buttons that the type property specifies. Each SkyConfirmButtonConfig object includes the following properties:
  • action — Specifies an identifier to return when users select the button to close the confirmation dialog. This is useful to determine which button users select. This property accepts string values.
  • text — Specifies the label for the button. This property accepts string values.
  • autofocus — (Optional.) Indicates whether to place focus on this button by default. This property accepts boolean values. (Default: false)
  • styleType — (Optional.) Specifies a SKY UX button class to apply to the button. The valid options are primary to format the primary action on the dialog, default to format a secondary action, and link to format an action that takes users to another page. This property accepts string values.
Indicates whether to preserve whitespace and new lines inside the confirm component. This property accepts boolean values. Specifies a SkyConfirmType enum that indicates how many buttons to include in the dialog. The valid options are:
  • Custom — Allows you to define your own buttons using the buttons property to specify an array of SkyConfirmButtonConfig objects. The integer value equivalent is 0.
  • OK — Displays one button with an OK label. The integer value equivalent is 1.
  • YesCancel — Displays two buttons with Yes and Cancel labels. The integer value equivalent is 2.
  • YesNoCancel — Displays three buttons with Yes, No, and Cancel labels. The integer value equivalent is 3.
Fires when users select an action to close the confirmation dialog. This event returns a SkyConfirmCloseEventArgs object with information about the button that users select. The SkyConfirmCloseEventArgs object includes the following property:
  • action — The identifier for the button that users selected to close the confirmation dialog.
Usage

A confirmation dialog focuses user attention on a specific question. It is appropriate for scenarios that meet the following criteria:

  • An action could be destructive or especially costly when performed by mistake.

  • Users must respond to a single question to proceed with the action.

  • The response to the question requires no input fields.

Content

The question portion of the dialog must contain only one question.

  • The text can be more than one sentence.

  • Do not use this component if the context of the question requires significant explanation, illustration, or dynamic feedback of downstream consequences.

The answer portion of the dialog answers the question or tells users what will happen. It can contain up to 3 choices represented as terminal actions (buttons) that should reiterate the action to take. For example, use "Delete" for the primary button text if the question is "Are you sure you want to delete this draft?"

  • A primary button triggers the recommended or most-common action.

  • A secondary button can trigger a less-common action.

  • A link button cancels the action.

  • Each button closes the dialog.
