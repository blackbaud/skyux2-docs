The tabs module renders tabs that divide large subsets of content on a page. It includes options to display buttons in the tab area and to make tabs closeable.

Use when

Use tabs when users require quick access to different slices of the same data.
Do use tabs to show different slices of the same data.

Use tabs when users need to complete different tasks on a page, and each task includes a significant amount of data.
Do use tabs to group content for different tasks.

Don't use when

Don't use tabs when a page already has a set of tabs. Change the workflow instead of using multiple sets of tabs.
Don't use multiple sets of tabs on a page.
Don't combine vertical tabs and horizontal tabs.

The tabs are likely to overflow the available horizontal space. This leads to a collapsed tab dropdown and a large amount of unused space in the tab bar. Use vertical tabs instead.
Don't use horizontal tabs when they are likely to overflow and collapse into a dropdown.

Count

When a tab displays a list, you can include the record count in the tab if seeing that number will meaningfully impact the user's behavior.

Add and close

If users need to work with a subset of tabs in a large tabset, let them add and close tabs to display only the relevant tabs.

If a new tab includes multiple options on what to display, use action buttons in the tab view to let users make selections.

Overflowing tabs

If the tabset includes too many tabs to fit in the tab bar's available horizontal space, the tabs collapse into a dropdown.

Page titles

If tabs are the main navigation on a page, display the tab header in the page title. Use the following format: <Page title> — <Company, product, or site identifier>. For example: Portfolio — Work center — Blackbaud. In addition, follow these guidelines for page titles:

  • Keep titles short.
  • Make titles unique.
  • Provide concise descriptions of each page's contents or purpose. Titles should stand on their own and make sense out of context.
  • Start with the most meaningful, differentiating terms for users to scan in browser tabs, window lists, and browser bookmarks.

For information about how to use the Angular Title service and the SKY UX title configuration property to specify the title, see the configuration reference.

Tabs are always the last (bottommost) content in their container. Don’t put additional content under tabs.
Don't put additional content under tabs.

The tab border extends the full width of the container. Don’t use left or right margins on the tab border, and ignore any padding that the container defines.
Do extend the tab border to the full width of the container.
Don't apply the container's padding to the tab border.

Tab content does not have additional left or right padding. The left and right gutters align with the gutters for content outside the tabs.
Don't apply additional left or right padding to tab content.
