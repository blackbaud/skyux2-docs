The status indicator classes provide icons to draw attention to status information that users need to address. Statuses indicate a state or condition at a particular time and fall into four categories: danger, info, success, and warning. For example, a status can indicate that a submission is late, submitted, or awaiting approval. Status indicator classes

UX guidelines
Status importance

If statuses are crucial to user tasks, use status indicators. However, if one status in an area is crucial, that does not mean all statuses are. For example, in a list of invoices with a mix of paid, unpaid, and overdue invoices, it's likely that only overdue invoices require status indicators because only they require immediate attention.

If statuses are not crucial to user tasks, use regular text.

Status indicator placement

Use status indicators inline to indicate that they are tied to specific pieces of data. For example, you can place status indicators in grid cells or cards. However, don't use status indicators in summary areas. To convey higher-level statuses in summary areas, use labels instead.
