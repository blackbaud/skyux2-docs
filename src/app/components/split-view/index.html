The split view component allows users to work through a list of items, while displaying each item's details in context. The split view component consists of two children components: the drawer and the workspace.

Use when

Use a split view if a user needs to work through a list of items with the intent of taking an action on each item in the list.

Don't use when

Don't use a split view if the user is likely to interact with a small number of items in the list. Use a regular list instead, following the flyout guidelines to determine whether to use a flyout or navigate to the selected record.

List type

Repeaters will be the most common component used for the list of items due to the flexible content layouts they enable.

Tree views can also be used if the items in the list are hierarchical. You may use different views in the workspace panel if the items at different levels of the tree are of different types.

While grids may technically be used, avoid them if possible. They do not scale well to the typically-small width of the navigation list.
Use caution when using grids as the list type as they do not perform well at narrow widths.

List manipulation

You may include a list toolbar so that users can filter, sort and search the list, or add new items to the list.

Automatically advance to the next item

If the user is likely to work through the list of items sequentially, automatically advance to the next item in the list when the user invokes the primary action on the record.

Set focus in the workspace

When the user selects a new item in the list, set focus on the first focusable element in the workspace so the user can immediately start working.

Change panel width

The user can change the width of the panels by drag-and-drop or keyboard interactions. If the navigation list content is wider than the panel, the panel will have a horizontal scrollbar.

Responsiveness

In smaller viewports the split view automatically switches from a side-by-side view to a panel-switching interaction where the user flips between the list and the workspace. The responsive workspace header should have iterator buttons so the user can move between items without having to go back to the list.

A split view should be the last item on a page; there should not be additional content below it.
Don't place content below the split view.
