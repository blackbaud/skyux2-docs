The character count component is an extension for text input fields with character limits.

Use when

Use character count when a user has a high likelihood of being impacted by a character limit. This could be because technical requirements make the limit low, or because user input is freeform and could be very long.

Do use character count on input fields with character limits that a user could realistically exceed.

Don't use when

Don’t use character count on inputs where the character limit is apparent or implied by the data type.

Don't use character count when the content has an expected or standard length.

Don’t use character count on inputs when exceeding the character limit is unrealistic. Use normal form validation to handle errors caused by character limit violations.

Don't use character count when the character limit is unlikely to be exceeded.
Do use normal form validation for edge cases where a high character limit is exceeded.
Input field Character count indicator Status indicator (danger)

The character count component is an extension of the form component, which defines the styles and appearance of the input field states.

For the validation message when the character limit is exceeded, use the following format:

Limit <input field label> to <n> characters.

The character count is always located above the right side of the input field. If the field label is long, it wraps to the next line without displacing the character count.
