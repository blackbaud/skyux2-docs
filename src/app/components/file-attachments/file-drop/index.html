The file drop component allows the user to drag and drop or select files to upload, and the file item displays information about an attachment.

Use when

Use file attachment when the user needs to upload multiple files or attach a file from a URL rather than the local device.
Do use file attachment to upload more than one file.

Use file attachment when the user needs to add metadata to the uploaded file.
Do use file attachment to add metadata about the uploaded file.

Don't use when

Don't use file attachment if the user needs to upload one local file. Use single file attachment instead.

Attach from URL

You can specify whether you want the user to be able to attach a file from a URL or only from the local device.

Validation

You can specify the parameters for a valid file, including file type, file size or a custom validation function.

File information

When the user attaches a file, summary information for that file appears in a card below the file attach input. By default the summary information includes file name, a delete button, and if the file is from a local drive also includes file size and a file thumbnail. You can optionally include inputs for the user to add metadata about the file.

Responsiveness

File attachment switches from a horizontal to a vertical layout in smaller viewports.

Guidelines

  • Form design

The sky-file-drop component provides an element that users can click to select files from their local drives or use as a drop zone to drag and drop files. The component can also display controls for users to provide hyperlinks to files on the web. You can leave the contents of the component blank to display the drop zone's default UI, or you can specify custom content to display instead.

When the SKY UX module initializes, it disables the ability to drag and drop files for the entire window. This prevents the browser from opening files that are accidentally dropped outside the target zone. If you implement your own file drop functionality outside of the file drop component, you can place the sky-file-drop-target CSS class on the element that receives drop events to exempt it from the drop exclusion rule.

The sky-file-item component displays summary information about files that users attach to forms. By default, the component displays file names, delete buttons, and fields for names and tags. For files from local drives, the component also displays file sizes and thumbnails.
