The single file attachment component allows the user to select a single local file to upload.

Use when

Use single file attachment when the user needs to upload one file to which they have local access.
Do use single file attachment to upload one file.

Don't use when

Don't use single file attachment if the user might need to link to a URL instead of uploading a local file. Use file attachment instead.

Don't use single file attachment when the user might need to upload more than one file. Use file attachment instead.
Don't use single file attachment if the user might upload multiple files.

Guidelines

  • Form design

The sky-file-attachment component provides an element that users can click to select a single file from their local drives or use as a drop zone to drag and drop a single file. For image files, the component will display a preview. For all files, a link is displayed that emits an event when clicked.

Provides a comma-delimited string literal of MIME types that users can drop or select ([acceptedTypes]="validFileTypes" or acceptedTypes="image/png,image/jpeg"). By default, any type of file is allowed. Specifies the maximum size in bytes for valid files. Specifies the minimum size in bytes for valid files. Provides a custom validation function ([validateFn]="validateFile"). By default, any type of file is allowed. Fires when the user selects or removes a file. Fires when the user clicks the file link.
