The checkbox component is a SKY UX-themed replacement for the HTML input element of type="checkbox". The value that is selected is driven through the ngModel attribute that you specify on the sky-checkbox element.

Indicates whether the checkbox is selected. This property accepts boolean values. Indicates whether the checkbox is disabled. This property accepts boolean values. Specifies an icon to display in place of the checkbox. This property accepts string values. If you want to use the icon checkbox group display in the demo, the sky-switch-icon-group class can be placed on the direct parent element of the checkboxes. Specifies an ID for the checkbox. If you do not specify an ID, an auto-incrementing integer generates unique IDs. Defines an ARIA label for the checkbox to support accessibility when the checkbox does not include a visible label. This property must be set if you are using an icon checkbox (see accessibility). This property accepts string values. Sets the checkbox's aria-labelledby attribute to support accessibility. The value should be the HTML element ID (without the leading #) of the element that labels the checkbox. If the checkbox does not include a visible label on the screen, use the label property instead. This property accepts string values. Specifies a name for a group of checkboxes. This property accepts string values. If you do not specify an ID, an auto-incrementing integer generates unique IDs. Specifies a type for a checkbox that uses the icon property. When users select the icon, this property changes its background color. The valid options correspond to the label types from the label component, and they include info, which creates a blue background, success, which creates a green background, warning, which creates an orange background, and danger, which creates a red background. This property accepts string values. It specifies an index for the checkbox. If the index is not defined, it is set to the position of the checkbox on load. Fires when users select or deselect the checkbox.

For a group of checkboxes, use fieldset and legend elements to provide semantic structure. For people using assistive technology, the fieldset element groups the checkboxes to indicate the number of items in the group, and the legend element provides a description for the group. For more information on how to use fieldset and legend, see WebAIM examples and guidance.

For icon checkboxes, the label or labelledby input must be set. For people using assistive technology, the visible icon may not convey sufficient information.
