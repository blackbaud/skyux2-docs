Data entry grids provide a spreadsheet-like user interface for entering large amounts of data.

Use when

Use data entry grids when:

  • Users have experience working with spreadsheets
  • Users will edit many values in one session
  • The dataset is large

Don't use when

Don't use data entry grids when:

  • Users need guidance for the task they are completing
  • Users will edit a small number of values in one session
  • The dataset is small

Disabled cells

You can disable columns of cells within the grid if users should not edit that data.

Multiselect

You can enable multiselect if users need to select multiple rows in the grid to take action on.

The SKY UX ag-Grid module provides styles, a service for default gridOptions, and SKY UX components for cell rendering and editing in ag-Grid. If you have never used ag-Grid before, the Angular getting started guide covers the basics. For more in depth information about ag-Grid and its capabilities, please reference the ag-Grid documentation.

SkyAgGridService is a service that provides methods to get ag-Grid gridOptions. The gridOptions can be overridden, and include registered SKY UX column types. Use the service when you are using ag-Grid to ensure your grid matches the standard SKY UX functionality.

These column types can be used by setting the ag-Grid column definition type to one of the following values.

In edit mode, a cell in an autocomplete column will render as a SKY UX autocomplete component. You can set any of the autocomplete component's properties by passing them in the column definition's cellEditorParams property. These params can be updated as other cell edits are made or provided dynamically based on other cell values. See the demo for an example.

In read-only mode, the cell will display the currently selected value's name property by default. If the autocomplete needs to show a different property or needs to be formatted in any way, you can define a valueFormatter on the column definition.

In edit mode, a cell in a date column will render as a SKY UX datepicker component. You can set any of the datepicker component's properties by passing them in the column definition's cellEditorParams property. These params can be updated as other cell edits are made or provided dynamically based on other cell values. See the demo for an example.

In read-only mode, the cell will display the currently selected date formatted as MM-DD-YYYY, or the date format of the locale passed to getGridOptions(). If you would like to overwrite this format, you can define a valueFormatter on the column definition. See the demo for an example.

In edit mode, a cell in a number column will render as an HTML number input. Only numbers can be entered.

In read-only mode, the cell will render as the number value.

In edit or read-only mode, a cell in a row selector column will render as a SKY UX checkbox component. It allows the user to select multiple rows, and adds a highlight to selected rows. The ag-Grid rowNode will be updated to reflect the selected state.

In edit mode, a cell in a text column will render as an HTML text input. Any characters can be entered.

In read-only mode, the cell will render as the string value.

Any SKY UX component can be made into a cell editor or cell renderer component. If you would like to use a component that does not have a column definition yet, please consider contributing it to the SKY UX ag-Grid module, or file an issue in the @skyux/ag-grid repo.

Passing cell editor parameters

You can pass inputs to the SKY UX components used in cell editors via the column definition's cellEditorParams property. The component properties are nested under skyComponentProperties. All component properties should be supported, and you can see the defined types by importing SkyCellEditorDatepickerProperties or SkyCellEditorAutocompleteProperties. cellEditorParams can also be a function that returns a param object for dynamic editing params. See the demo for examples of using an object or a function.

ag-Grid styles

To add the SKY UX styles for ag-Grid to your SPA, include "@skyux/ag-grid/css/sky-ag-grid.css" in the styles section of your skyuxconfig.json. For editable grids, include the class sky-ag-grid-editable on the ag-grid-angular element.
