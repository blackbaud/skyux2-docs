The list component displays a SKY UX-themed list of data in a consistent, flexible way.

Several related components are available for use within the list component, including the list filters module, the list paging component, the list toolbar component, the list view checklist component, and the list view grid component.

Specifies the data to display. Must provide data or dataProvider. Acceptable values: Observable Array or Array. To update data in your view outside of a dataProvider, you must use an observable instead of a static array). Specifies a data provider used to obtain data to display. Must provide data or dataProvider. Acceptable values: extends ListDataProvider. To extend ListDataProvider, provide a get function that accepts a ListDataRequestModel as an argument and returns an Observable ListDataResponseModel , and a count function that returns an Observable number that indicates how many total records are in the list. Specifies the total number of items for the initial data set when initialized. When used in conjunction with data and dataProvider, it allows an initial data to be set with the need to call the dataProvider. Specifies a function to apply as a global sort on the list. Acceptable value: (searchText: string) => Observable Array or Promise Array. Specifies a set of fields to sort by. If array of fields then sorted by order of array. Acceptable values: ListFieldSelectorModel, Observable ListFieldSelectorModel, Array of ListFieldSelectorModel, or Observable Array of ListFieldSelectorModel. Where ListFieldSelectorModel is an object with the fieldSelector property (a string that specifies the property of the data that is actively being sorted upon) and the descending property (a boolean that indicates the active sort is descending). Specifies a set of selected IDs to use with a checklist view. Acceptable values: Observable Array of strings or Array of strings. Specifies a set of filters to apply to list data. These filters also create a filter summary if the sky-list-filter-summary component is included. Acceptable values: Array of ListFilterModel. When using a list view that supports item selection, the list emits this event with a Map string, boolean object that indicates which data entries are selected. The list emits this event with a Array of ListFilterModel with the new filters applied to the list.
