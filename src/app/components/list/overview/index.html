The list component displays a SKY UX-themed list of data in a consistent, flexible way.

Several related components are available for use within the list component, including the list filters module, the list paging component, the list toolbar component, the list view checklist component, and the list view grid component.

Specifies a set of filters to apply to list data. These filters also create a filter summary if the sky-list-filter-summary component is included. Acceptable values: Array of ListFilterModel. Specifies the data to display. The list component requires this property or the dataProvider property. For checklist or multiselect grids, each row requires an id property to manage selected items with the selectedIds input property and the selectedIdsChange event. If you do not provide an id, the list automatically generates one. This property accepts Observable Array or Array values. (To update data in your view outside of a dataProvider, you must use an observable instead of a static array.) Specifies a data provider to obtain the data to display. The list component requires this property or the data property. Acceptable values: extends ListDataProvider. To extend ListDataProvider, provide a get function that accepts a ListDataRequestModel as an argument and returns an Observable ListDataResponseModel , and a count function that returns an Observable number that indicates how many total records are in the list. For lists that use dataProvider instead of data, consumers are responsible for managing all ListDataRequestModel properties. For examples of custom searching, filters, and paging, see the demo code below. Specifies the total number of items for the initial data set when initialized. When used in conjunction with data and dataProvider, it allows an initial data to be set with the need to call the dataProvider. Specifies a function to apply as a global sort on the list. Acceptable value: (searchText: string) => Observable Array or Promise Array. Specifies a set of IDs for the items to select in a checklist or multiselect grid. The IDs match the id properties of the data objects. Items with IDs that are not included are de-selected in the checklist or multiselect grid. This property accepts an observable array of strings or array of strings. Specifies a set of fields to sort by. If array of fields then sorted by order of array. Acceptable values: ListFieldSelectorModel, Observable ListFieldSelectorModel, Array of ListFieldSelectorModel, or Observable Array of ListFieldSelectorModel. Where ListFieldSelectorModel is an object with the fieldSelector property (a string that specifies the property of the data that is actively being sorted upon) and the descending property (a boolean that indicates the active sort is descending). The list emits this event with a Array of ListFilterModel with the new filters applied to the list. When using a list view that supports item selection, the list emits this event with a Map string, boolean object that indicates which data entries are selected.
