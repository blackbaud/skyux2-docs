The button classes create buttons to trigger actions from within an interface. For information about styles and implementation, see the buttons and links guidelines.

Usage

The primary button triggers the most important or most frequently used action on a page or the progressive terminal action on a modal (such as Save or Done). Pages and modals should only include one primary button to ensure that it draws user attention. For information about how to use primary actions, see the primary action guidelines.

Code
<button type="button" class="sky-btn sky-btn-primary">Primary</button>
Usage

Secondary buttons trigger less important or less frequently used actions. Since secondary buttons are less important than primary buttons, SKY UX reduces the visual weight to make them less attention-drawing than the primary buttons.

Code
<button type="button" class="sky-btn sky-btn-default">Secondary</button>
Usage

Link buttons trigger actions that take users to other pages or actions that end tasks or exit modals (such as Cancel).

Code
<button type="button" class="sky-btn sky-btn-link">Link button</button>
Usage

Inline link buttons trigger the same type of actions as link buttons, but the styling is altered to fit inline.

Code
<button type="button" class="sky-btn sky-btn-link-inline">Inline link</button>
Usage

The disable class disables all button types. The most common use case is to disable actions until users complete prerequisite tasks.

Code
class="sky-btn-disabled"
Button text

For buttons with only one possible direct object (such as Add buttons in record list toolbars), use the following format for button text:

<Verb>

For buttons with multiple possible direct objects (such as Add alert buttons in page summary toolbars), use the following format for button text:

<Verb> <direct object>

Additional button text guidelines:

  • Use sentence-style capitalization.
  • Make the direct object singular or plural based on whether the action lets users act on one or multiple items.
  • Don't include articles. For example, use Add constituent rather than Add a constituent.
  • Don’t append extra words such as “details.” For example, use Edit constituent rather than Edit constituent details.
