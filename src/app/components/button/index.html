The button classes create buttons to trigger actions from within an interface. For information about styles and implementation, see the buttons and links guidelines.

Classes
Usage

The primary button triggers the most important action on a page, the most frequently used action on a page, or the progressive terminal action on a dialogue (such as 'Save' or 'Done'). Pages and dialogues should only include one primary button to ensure that it draws user attention. For information about how to use primary actions, see the primary action guidelines.

Code
<button type="button" class="sky-btn sky-btn-primary">Primary</button>
Usage

Secondary buttons trigger less important or less frequently used actions. Since secondary buttons are less important than primary buttons, SKY UX reduces the visual weight to make them noticeable but not attention-drawing.

Code
<button type="button" class="sky-btn sky-btn-default">Secondary</button>
Usage

Link buttons can trigger actions that take users to other pages. They can also trigger actions that end tasks or exit modals. For example, you can use this option to create a Cancel button.

Code
<button type="button" class="sky-btn sky-btn-link">Link button</button>
Usage

Inline link buttons trigger the same type of actions as link buttons, but the styling is altered so that the buttons fit inline.

Code
<button type="button" class="sky-btn sky-btn-link-inline">Inline link</button>
Usage

You can disable all button types by applying an additional class. The most common case is to disable actions until users complete the prerequisite tasks.

Code
class="sky-btn-disabled"
UX guidelines
Button text

Use the following format for button text:

<Verb> <direct object>

Additional button text guidelines:

  • Use sentence case.
  • The direct object can be singular or plural depending on whether the action lets you act on one or multiple items.
  • Don’t append extra words such as “details.” For example, use “Edit constituent” rather than “Edit constituent details.”
