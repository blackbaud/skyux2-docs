The flyout service launches a container over a page's content to display large quantities of supplementary information related to a task. The SkyFlyoutService provider lauches flyouts and provides a common look-and-feel. The SkyFlyoutInstance class watches for and triggers flyout events.

The SkyFlyoutService dynamically generates the flyout component and appends it directly to the document's body element. Opens a flyout and displays the specified component. The optional config parameter is of type SkyFlyoutConfig and is passed to the specified component's constructor. This method returns the flyout instance. Closes the flyout. This method also removes the flyout's HTML elements from the DOM. Specifies an array of custom providers to pass to the component's constructor. Sets the flyout's role attribute to support accessibility. Sets the flyout's aria-labelledby attribute to support accessibility. The value should be the HTML element ID (without the leading #) of the element that labels the flyout. Sets the flyout's aria-describedby attribute to support accessibility. The value should be the HTML element ID (without the leading #) of the element that describes the flyout. Typically, this is the text on the flyout but does not include items that users interact with such as buttons and forms. Closes the flyout instance and emits its closed event. Specifies the instance of the component to display in the flyout. An event that the flyout instance emits when it is closed.

Follow these guidelines to ensure that people can use flyouts with assistive technology or keyboards.

  • When a flyout opens, place focus on the first focusable item. If the flyout does not include focusable items, then place focus on the flyout container element.
  • While a flyout is open, limit focus to elements in the flyout. Do not move focus back to the initiating page behind the flyout. On the last focusable item, the Tab key should return focus to the first focusable item. And on the first focusable item, the Shift+Tab keys should switch focus to the last focusable item.
  • When a flyout closes, return focus to the element that had focus before the flyout opened. This is usually the element that opened the flyout.

Flyouts are appropriate for the following scenarios:

  • Users need to open records in lists and return to the list views after quick tasks on the record pages.
  • Users need to drill into analytical insights and quickly research those insights without creating additional workflows.
  • Users need to drill into child records and retain the context of the parent record.

Flyouts are not appropriate for the following scenarios:

  • Users need to navigate from one record to another record of the same type.
  • Users need to navigate from a child page to its parent page.
  • A detail panel is already open.
  • A page would not make sense without the detail panel.
  • Users need to view a small quantity of supplementary information.
  • Users need to view items such as system-generated reports that take a long time to load.
  • Users need to view supplementary information from external sites.
  • Users need to open tasks that always require the full screen view of details.
  • The screen size is 480px or smaller.
