The flyout service launches a container that lays over the page content to display supplementary information related to a task. The SkyFlyoutService provider lauches the flyouts with a common look-and-feel for inner content. The SkyFlyoutInstance class allows you to watch for and trigger flyout events.

The SkyFlyoutService dynamically generates the flyout component and appends it directly to the document's body element. Opens a flyout to display the specified component. The optional config parameter is of type SkyFlyoutConfig and is passed to the provided component's constructor. This method returns the flyout instance. Closes the flyout. This method also removes the flyout's HTML elements from the DOM. Specifies an array of custom providers to pass to the component's constructor. Sets the flyout's role attribute for accessibility support. Sets the flyout's aria-labelledby attribute for accessibility support. The value should be the HTML element ID (without the leading #) for the element that labels the flyout. Sets the flyout's aria-describedby attribute for accessibility support. The value should be the HTML element ID (without the leading #) for the element that describes the flyout. Typically, this is the text on the flyout, not including items that users interact with such as buttons or forms. Closes the flyout instance and emits the closed event of the flyout instance. Specifies the instance of the component to display in the flyout. An event that the flyout instance emits when it is closed.

Follow these guidelines to ensure that people can use flyouts with assistive technology or keyboards.

  • When a flyout opens, place focus on the first focusable item. If the flyout does not include focusable items, then place focus on the flyout container element.
  • While a flyout is open, limit focus to elements in the flyout when using Tab key to navigate. Do not move focus back to the initiating page behind the flyout. On the last focusable item, the Tab key should return focus to the first focusable item. Conversely, on the first focusable item, the Shift+Tab keys should switch focus to the last focusable item.
  • When a flyout closes, return focus to the element that had focus before the flyout opened. This is usually the element that opened the flyout.

Flyouts are appropriate for the following scenarios:

  • Users need to open records in lists and return to the list views after quick tasks on the record pages.
  • Users need to drill into analytical insights and quickly glance into what makes up those insights without spawning additional workflows.
  • Users need to drill into child records and retain the context of the parent record.

Flyouts are not appropriate for the following scenarios:

  • Users need to navigate from one record to another instance of the same record type.
  • Users need to navigate from a child page to its parent page.
  • A detail panel is already open.
  • A page would not make sense without the detail panel open.
  • You only need to display a small amount of supplementary information.
  • You need to display items that take a long time to load such as system-generated reports.
  • You need to link to external sites
  • Users need to open tasks where they always need to see the full screen view of the details.
  • The screen size is 480px or smaller.
