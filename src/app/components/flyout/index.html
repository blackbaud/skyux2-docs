The flyout service launches a container that lays over the page content to show larger amounts of supplementary information related to the user's task. The SkyFlyoutService provider lauches the flyouts with a common look-and-feel for given inner content. The SkyFlyoutInstance class allows you to watch for and trigger flyout events.

The flyout component is generated dynamically using the SkyFlyoutService and appended directly to the document's body element. Opens a flyout using the specified component as the contents of the flyout. The optional config parameter is of type SkyFlyoutConfig, which will be passed to the provided component's constructor. This method returns the flyout instance. Closes the currently opened flyout. This method also removes the flyout's respective HTML elements from the DOM. An array of custom providers to be passed to the content component's constructor. Sets the role attribute for the flyout for accessibility support. Sets the aria-labelledby attribute for the flyout for accessibility support. The value should be an HTML element ID (without the leading #) pointing to the element that labels your flyout. Sets the aria-describedby attribute for the flyout for accessibility support. The value should be an HTML element ID (without the leading #) pointing to the element that describes your flyout. Typically, this will be the text on your flyout, but does not include something the user would interact with, like buttons or a form. Closes the flyout instance. The closed event of the flyout instance will be emmitted. The instance of the component that makes up the contents of the opened flyout. An event that is emitted by the flyout instance when it is closed.

Follow these guidelines to ensure that people can use flyouts with assistive technology or keyboards.

  • When a flyout opens, focus should be on the first focusable item in the flyout. If the flyout does not include focusable items, then focus should be on the flyout container element.
  • While a flyout is open, focus should remain on elements in the flyout when using Tab key to navigate. Focus should not move back to the initiating page behind the flyout. At the end of the flyout, the Tab key should return focus to the first focusable item in the flyout. Conversely, at the top of the flyout, the Shift+Tab keys should switch focus to the last focusable item in the flyout.
  • When a flyout closes, focus should return to the element that had focus before the flyout opened. Usually this is the element that opened the flyout.

Flyout

A flyout is appropriate for content that meet any of the following criteria:

  • When opening records in lists to allow a user to return to the list view after completing a quick task on the record page
  • When drilling into an analytical insight to allow a user to quickly glance into what makes up that insight without spawning another workflow
  • When drilling into child records to allow a user to retain the context of the parent record

A flyout is not appropriate for content that meet any of the following criteria:

  • When navigating from one record to another instance of the same record type
  • When navigating from a child page back to the parent page
  • When there is already a detail panel open
  • When the page would not make sense without the panel open
  • When there is only a small amount of supplementary information to display
  • When displaying items that need a considerable amount of time to load, such as a system generated report
  • When linking to external sites
  • When opening a task where the user will always need to see the full screen view of the details
  • When the screen size is 480px or smaller
