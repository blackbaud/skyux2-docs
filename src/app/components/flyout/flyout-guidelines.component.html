Usage

A flyout is a container that lays over page content to display large amounts of supplementary information related to user tasks. A flyout can contain lists, record views, graphs, or bespoke information displays.
When to use a flyout

Use a flyout to open records in a list so that users can return to the list view after completing quick tasks on the record pages.

Use a flyout to drill into an analytical insight so that users can quickly glance into what makes up that insight without spawning another workflow.

Use a flyout to drill into child records so that users can retain the context of the parent record. The flyout only covers part of the current page, so users can refer back to it for context.
When not to use a flyout

Do not use a flyout to navigate from one record to another instance of the same record type. For example, if users on a constituent record select a link to a different constituent record, navigate to the new record. Do not open it in a flyout.

Do not use a flyout to navigate from a child page to its parent page. For example, if users on a gift record select a link to return to a constituent record, navigate to a full-page view of the constituent record. Do not open it in a flyout.

Do not use a flyout when another flyout is already open. If users select a link in a flyout, close the flyout and navigate to the new link location. In other words, limit flyouts to a single drill-in. On further drilling, open a full page.

Do not use a flyout on pages that don't make sense when it closes. For example, on a page to enter grades for a class of students, displaying the names of students doesn't provide value without the grade entry form, so don't use a flyout to display the grade entry form.

Do not use a flyout to display a small amount of additional information. The flyout is for large amounts of supplementary information. To display smaller amounts of information, use a popover component or an expandable repeater.

Do not use a flyout to display items that need a long time to load such as system-generated reports. Instead, let users continue to use the system, and use an alert component to let them know when the content is available.

Do not use a flyout to link to external sites. Open external sites in new tabs. For example, when users select a constituent address, a Google map of that address open in a new tab.

Do not use a flyout for tasks where users always need a full-screen view of details, such as expanded views of tiles. Navigate to a full page instead.
Interactions
Background page remains active

The background page is active behind the flyout panel. Selecting a link for another record on the background page updates the flyout to load the new record. This is different than selecting a link inside the flyout, which navigates away from the current page.

Selecting a non-interactive element on the background page closes the flyout. On touch devices, touching a non-interactive element on the background page does not close the flyout because users need to be able touch the background page in order to scroll.
Independent scrolling

The flyout and the page scroll independently. If users touch, hover, or focus on the background page when scrolling, only the background content scrolls. Similarly, if users touch, hover, or focus on the flyout when scrolling, only the flyout scrolls. This allows users to adjust the content in the flyout without losing any context on the current page.
Drag to resize

Users can select the flyout's left border and drag it to adjust the size of the flyout. They can adjust it anywhere from 320px to the full screen size. If users expand the flyout to the full screen size, it automatically slides back 20px to the right to keep a portion of the page in view. The default size of the flyout is 50 percent of the screen or 320px, whichever is larger. When users resize the panel, the system remembers their preferred size globally.
Help panel

The flyout covers the omnibar, but the help panel remains visible in the upper right corner of the screen. The help panel should know that the flyout is open and provide appropriate guidance based on the context.
Modal dialog behavior

When users perform actions in the flyout that launch modal dialogs, the modal dialogs should cover the entire page.
Options
Next/previous row buttons

When a flyout opens records in a list for users to perform quick tasks on record pages, buttons in the flyout toolbar allow users to access the next record or previous record. The next button opens the record in the next row that shares a type with the list. For example, in a gifts list, if users open the flyout on the constituent record for the first gift in the list, clicking on the next button will open the gift record of the second row in the list. When the next/previous option is turned on, there is a visual indicator on the row in the list to show which row is currently selected.

When the flyout is used in a list to view records, there are buttons in the panel toolbar to go to the next record or the previous record. Clicking on the next button will open the record in the next row that shares a type with the list. For example, in a gifts list, if a user has the flyout open on the constituent record for the first gift in the list, clicking on the next button will open the gift record of the second row in the list. When the next/previous option is turned on, there is a visual indicator on the row in the list to show which row is currently selected.
List toolbar

When viewing a list in the flyout, the toolbar should give users the option to leave the "preview" list and create a new list for further manipulation. Clicking on the create list button will navigate to a new page containing the new list.
Responsiveness
Mobile behavior

The flyout should only be used when the screen is larger than 480px because it covers too much of the screen at smaller resolutions, thus losing much of its value. For smaller screen sizes, the system should navigate directly to the appropriate record or list.
