The dropdown component renders a button that displays a menu when users click it. The menu displays a set of actions or functions. Do not use the dropdown component as a select input for users to set the menu's value to one of the options. If you need a select input, see the options to select items from lists.

Specifies the type of button to render as the dropdown's trigger element. To display a button with text and a caret, specify select and then enter the button text in a sky-dropdown-button element. To display a round button with an ellipsis, specify context-menu. And to display a button with a Font Awesome icon, specify the icon's class name. For example, to display the fa-filter icon, specify filter. Specifies a background color for the dropdown button. If unspecified, the default value sets the background color to sky-color-white. To change the background color to sky-color-blue, specify primary. Indicates whether to close the dropdown when users click away from the menu. This property accepts boolean values. Specifies an accessibility label to provide a text equivalent for screen readers when the dropdown button has no text. Specifies a title to display in a tooltip when users hover the mouse over the dropdown button. Provides an observable to send commands to the dropdown. The commands should respect the SkyDropdownMessage type (see below). Sets the dropdown menu's aria-labelledby attribute to support accessibility. The value should be the HTML element ID (without the leading #) of the element that labels the dropdown menu. Sets the dropdown's role attribute to support accessibility. For information about ARIA roles, see the WAI-ARIA roles model. Indicates whether to use the browser's native focus function when users navigate through menu items with the keyboard. To disable the native focus function, set this property to false. For example, if you want to interact with the dropdown menu but keep the keyboard focus on a different element, set this property to false. Fires when the dropdown menu's active index or selected item changes. This event provides an observable to emit changes, and the response is of type SkyDropdownMenuChange (see below). Sets the message to one of the following SkyDropdownMessage types:
  • SkyDropdownMessageType.Open — Opens the dropdown menu.
  • SkyDropdownMessageType.Close — Closes the dropdown menu.
  • SkyDropdownMessageType.FocusTriggerButton — Puts focus the dropdown button.
  • SkyDropdownMessageType.FocusNextItem — Puts focus on the next item in the dropdown menu.
  • SkyDropdownMessageType.FocusPreviousItem — Puts focus on the previous item in the dropdown menu.
  • SkyDropdownMessageType.Reposition — Repositions the dropdown next to the dropdown button. This is useful for when the dropdown's width and height change while open.
This property is deprecated, and we recommend against using it. It specifies the user action that opens the dropdown, and the valid options are click or hover. We recommend the default click value because the hover option can pose accessibility issues for users on touch devices such as phones and tablets. Indicates the active menu index. Indicates the selected item in the menu. The value of the field is of type SkyDropdownItemComponent.
Dropdown menu action text

If the context menu is tied to the direct object of a context menu item, use the following format for the action wording:

<Verb>

For a dropdown button with one type of action, use the following format:

Button text: <Verb>

Menu item: <Direct object>

For a dropdown button with more than one type of action, use the following format:

Button text: More

Menu item: <Verb> <Direct object>

Additional button text guidelines:

  • Use sentence case.
  • The direct object can be singular or plural depending on whether the action lets you act on one or multiple items.
  • Don’t append extra words such as “details.” For example, use “Edit constituent” rather than “Edit constituent details.”
