The dropdown component renders a button that displays a menu when users click it. The menu displays a set of actions or functions. Do not use the dropdown component as a select input for users to set the menu's value to one of the options. If you need a select input, see the options to select items from lists.

Specifies the horizontal alignment of the dropdown menu in relation to the dropdown button. Available values are left, right, and center. Specifies a background color for the dropdown button. Available values are default and primary. These values set the background color from the secondary and primary button classes respectively. Specifies the type of button to render as the dropdown's trigger element. To display a button with text and a caret, specify select and then enter the button text in a sky-dropdown-button element. To display a round button with an ellipsis, specify context-menu. And to display a button with a Font Awesome icon, specify the icon's class name. For example, to display the fa-filter icon, specify filter. This property accepts string values. Indicates whether to disable the dropdown button. This property accepts boolean values. Indicates whether to close the dropdown when users click away from the menu. This property accepts boolean values. Specifies an accessibility label to provide a text equivalent for screen readers when the dropdown button has no text. This property accepts string values. Provides an observable to send commands to the dropdown. The commands should respect the SkyDropdownMessage type. Specifies a title to display in a tooltip when users hover the mouse over the dropdown button. This property accepts string values. This property is deprecated. We recommend against using it. It specifies how users interact with the dropdown button to expose the dropdown menu, and the available values are click and hover. We recommend the default click value because the hover value can pose accessibility issues for users on touch devices such as phones and tablets. If you choose to use the deprecated hover value anyway, we recommend that you not use it in combination with the title property. Sets the dropdown menu's aria-labelledby attribute to support accessibility. The value should be the HTML element ID (without the leading #) of the element that labels the dropdown menu. This property accepts string values. Specifies an ARIA role for the dropdown menu to support accessibility by indicating how the dropdown menu functions and what it controls. The ARIA role indicates what the dropdown menu represents on the web page. For information about ARIA roles, see the WAI-ARIA roles model. This property accepts string values. Indicates whether to use the browser's native focus function when users navigate through menu items with the keyboard. To disable the native focus function, set this property to false. For example, to let users interact with the dropdown menu but keep the keyboard focus on a different element, set this property to false. This property accepts boolean values. Fires when the dropdown menu's active index or selected item changes. This event provides an observable to emit changes, and the response is of the SkyDropdownMessage type. Specifies an ARIA role for the dropdown menu item to support accessibility by indicating how the dropdown menu item functions and what it controls. The ARIA role indicates what the dropdown menu item represents on the web page. For information about ARIA roles, see the WAI-ARIA roles model. This property accepts string values. Sets SkyDropdownMessage to one of the following types:
  • SkyDropdownMessageType.Close — Closes the dropdown menu.
  • SkyDropdownMessageType.FocusNextItem — Puts focus on the next item in the dropdown menu.
  • SkyDropdownMessageType.FocusPreviousItem — Puts focus on the previous item in the dropdown menu.
  • SkyDropdownMessageType.FocusTriggerButton — Puts focus on the dropdown button.
  • SkyDropdownMessageType.Open — Opens the dropdown menu.
  • SkyDropdownMessageType.Reposition — Repositions the dropdown menu next to the dropdown button. This is useful for when the dropdown menu's width and height change while it is open.

Dropdown menu action text

If the context menu is tied to the direct object of a context menu item, use the following format for the action wording:

<Verb>

For a dropdown button with one type of action, use the following format:

Button text: <Verb>

Menu item: <Direct object>

For a dropdown button with more than one type of action, use the following format:

Button text: More

Menu item: <Verb> <direct object>

Additional button text guidelines:

  • Use sentence-case capitalization.
  • The direct object can be singular or plural depending on whether the action acts on one or multiple items.
  • Don’t append extra words such as “details.” For example, use “Edit constituent” rather than “Edit constituent details.”
