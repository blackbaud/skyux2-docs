This grid component displays data in a consistent and flexible way. The grid should contain sky-grid-column components that specify column information. The sky-grid component can provide a view of a searchable, paginated list component when used within the sky-list component.

Specifies columns and column properties for the grid. This property accepts an array of SkyGridColumnModel values. Specifies the data for the grid. Each item requires an id and a property that maps to the field or id property of each column in the grid. This property accepts an array of any values. Indicates whether to enable the multiselect feature to display a column of checkboxes on the left side of the grid. You can specify a unique ID with the multiselectRowId property, but multiselect defaults to the id property on the data object. To include options to select and clear all multiselect checkboxes, we recommend the list view grid component. This property accepts boolean values. Specifies how the grid fits to its parent. The valid options are width, which fits the grid to the parent's full width, and scroll, which allows the grid to exceed the parent's width. If the grid does not have enough columns to fill the parent's width, it always stretches to the parent's full width. This property accepts string values. Indicates whether to display a toolbar with the grid. This property accepts boolean values. Specifies the height of the grid. This property accepts number values. Specifies text to highlight within the grid. Typically, this property is used in conjunction with search. The property accepts string values. Provides an observable to send commands to the grid. The commands should respect the SkyGridMessage type. (See below.) Specifies a unique ID that matches a property on the data object. By default, this property uses the id property. This property accepts string values. Specifies the ID of the row to highlight. The ID matches the id property of the data object. Typically, this property is used in conjunction with the flyout component to indicate the currently selected row. This property accepts string values. Specifies the columns to display in the grid based on the id or field properties of the columns. If no columns are specified, then the grid displays all columns. This property accepts an array of string values. Specifies a set of IDs for the rows to select in a multiselect grid. The IDs match the id properties of the data objects. Rows with IDs that are not included are de-selected in the grid. This property accepts an array of string values. Specifies a unique key for the UI Config Service to retrieve stored settings from a database. The UI Config Service saves configuration settings for users and returns selectedColumnIds to preserve the columns to display and the preferred column order. This property accepts string values. For more information about the UI Config Service, see the sticky settings documentation Displays a caret in the column that was used to sort the grid. This is particularly useful when you programmatically sort data and want to visually indicate how the grid was sorted. This property accepts a ListSortFieldSelectorModel value to represent the current sort field and the sort order. Specifies the width of the grid in pixels. This property accepts number values. Sets the message to one of the following SkyGridMessage types:
  • SkyGridMessageType.ClearAll — Clears the multiselect checkboxes for all rows in the grid.
  • SkyGridMessageType.SelectAll — Selects the multiselect checkboxes for all rows in the grid.
Represents the current sort field. This property accepts string values. Indicates whether to sort in descending order. The caret that visually indicates the sort order points down for descending order and up for ascending order. This property accepts boolean values. Fires when the width of a column changes. Emits an array of SkyGridColumnWidthModelChange. Fires when the selection of multiselect checkboxes changes. Emits an array of IDs for the selected rows based on the multiselectRowId property that the consumer provides. Fires when the columns to display in the grid change or when the order of the columns changes. The event emits an array of IDs for the displayed columns that reflects the column order. Fires when the active sort field changes. Emits a ListSortFieldSelectorModel. Specifies a description for the column. This property accepts string values. Specifies text to display in the column header. This property accepts string values. Specifies the horizontal alignment of the column's data and header. The valid options are center, left, and right. Specifies the property to retrieve cell information from an entry on the grid data array. You must provide either the id or field property for every column, but do not provide both. If id does not exist on a column, then field is the entry for the grid selectedColumnIds array. This property accepts string values. Indicates whether the column is initially hidden when grid selectedColumnIds are not provided. This property accepts boolean values. Specifies a unique ID for the column. You must provide either the id or field property for every column, but do not provide both. If field does not exist on a column, then the id property retrieves cell information from an entry on the grid data array. This property accepts string values. Specifies a template to display inside an inline help popup for this column. This property accepts a SkyPopoverComponent. Indicates whether the column sorts the grid when users click the column header. This property accepts boolean values. Indicates whether the column is locked. The intent is to display locked columns first on the left side of the grid. If set to true, then users cannot drag the column to another position and or drag other columns before the locked column. This property accepts boolean values. Specifies a search function to apply for the specific column. By default, the column executes a string compare on the column data. Acceptable values: (data: any, searchText: string) => boolean Specifies a template for a column. This can be assigned as a reference to the template attribute, or it can be assigned as a child of the template element inside the sky-grid-column component. The template has access to the value variable, which contains the value passed to the column, and the row variable, which contains the entire row data. Specifies the width of the column in pixels. If undefined, the column width is evenly distributed. This property accepts number values.
