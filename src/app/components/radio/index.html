The radio button component renders a SKY UX-themed replacement for an HTML input element with type="radio". When users select a radio button, its value is driven through an ngModel attribute that you specify on the sky-radio element or the parent sky-radio-group element. The sky-radio-label component specifies labels for radio buttons, and the sky-radio-group component organizes radio buttons into a group. The sky-radio-group component is required for radio buttons on Angular reactive forms, and we recommend using it with all radio buttons. The sky-radio-group component wraps sky-radio components to organize radio buttons into a group. It is required for SKY UX radio buttons on Angular reactive forms to manage the selected values and keep the forms up-to-date. We recommend using the sky-radio-group component with all radio buttons.

For a group of radio buttons, use fieldset and legend elements to provide semantic structure. For people using assistive technology, the fieldset element groups the radio buttons to indicate the number of items in the group, and the legend element provides a description for the group. For more information on how to use fieldset and legend, see WebAIM examples and guidance.

For icon radio buttons, the label or labelledby input must be set. For people using assistive technology, the visible icon may not convey sufficient information.

According to accessibility guidelines, focus on a group of radio button elements should only ever focus the currently selected radio button or if one isn't selected, the first/last button depending on the direction navigated from. So the tabIndex of a group of radio buttons should be placed on the sky-radio-group element.
