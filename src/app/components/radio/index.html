The radio button component is a SKY UX-themed replacement for the HTML input element of type="radio". The value that is selected is driven through the ngModel attribute specified on the input element. Indicates whether the radio button is selected. This property accepts boolean values. This property accepts string values. Specifies a name for a collection of radio buttons. This property accepts string values. Specifies a type for the radio button. This property accepts string values. Specifies the index of the radio button. If not defined, it will be set to the position of the radio button on load. Specifies a value for the radio button. Fires when the selected radio button changes.

The radio group component is a way to use the radio button component with Angular reactive forms. The selected value is instead managed by the radio group component which keeps the reactive form up to date.

Overwrites the name property of the radio button components within the group. Overwrites the selected value of the radio group.

For a group of radio buttons, use fieldset and legend elements to provide semantic structure. For people using assistive technology, the fieldset element groups the radio buttons to indicate the number of items in the group, and the legend element provides a description for the group. For more information on how to use fieldset and legend, see WebAIM examples and guidance.
