The radio button module ceates a small set of selection inputs and allows users to select one option only.

Use radio buttons when users must choose one option from a set of up to five. Use radio buttons in modals where users confirm decisions before changes take effect.

Use icon radio button groups when icons clearly communicate the options and fit under a single label.

Use radio buttons to enable conditional inputs. For options that enable one or two inputs, disable the inputs until users select the radio buttons. For options that enable more inputs, hide the inputs until users select the radio buttons. If all options in a radio button group enable conditional inputs or significantly change the workflow, consider using action buttons instead.

Don’t use radio buttons when users must choose from more than five items. Use a different selection input instead.

Don’t use radio buttons for two options that are exact opposites. Use a checkbox instead.

Don't use radio buttons when user selections significantly alter the workflow. Use action buttons instead, and place them at the beginning of the task.

Don’t use icon radio buttons when icons don't clearly communicate options. Use radio buttons with text labels to provide context.

Don’t use ungrouped icon radio buttons. Use icon radio button groups to connect the options visually.

Text label

When radio buttons are in grid columns, text labels are not required if column headings provide the necessary context. Use text labels for all other states.

Icon radio button type

Icon radio buttons can use background and border color that match status indicator types such as success or danger.

Radio buttons may or may not use a preceeding label to provide context for the options. Unlike text input labels, this label may use verbs or questions. If you do not include a label, provide context for the consequences of the choices in the radio button text labels.

Icon radio buttons use the FontAwesome 4.7 icon library. For a list of common icons, see the SKY UX icon dictionary. Icon radio buttons must use a label or labelledBy property for accessibility.

Arrange traditional radio buttons vertically to make it easy to see all options, scan their labels, and see the selected option.

Use the form classes to separate radio button groups from other form elements.

  • Form design

The sky-radio-group component organizes radio buttons into a group. IT is required for radio buttons on Angular reactive forms, and we recommend using it with all radio buttons. On Angular forms, the component manages the selected values and keeps the forms up-to-date. When users select a radio button, its value is driven through an ngModel attribute that you specify on the sky-radio-group element.

For a group of radio buttons, use the fieldset and legend elements to provide semantic structure. For people using assistive technology, the fieldset element groups the radio buttons to indicate the number of items in the group, and the legend element provides a description for the group. For more information about how to use fieldset and legend, see WebAIM examples and guidance.

Also for a group of radio buttons, use the tabIndex property to specify an index for all radio buttons in the group. This places focus on the currently selected radio button. If no radio button is selected, it places focus on the first or last button depending on how users navigate to the radio button group.

For icon radio buttons, use the label or labelledby input. For people using assistive technology, the visible icon may not convey sufficient information.
