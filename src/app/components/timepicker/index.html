The timepicker module contains directives to let users select times. It is a SKY UX-themed replacement for the HTML input element with type="time". The value that users select is driven through the ngModel attribute specified on the input element.

Use when

Use timepicker when users need to specify a time of day.
Do use timepicker when users need to input a 12-hour or 24-hour time.

Use two timepickers when users need to specify a start time and stop time.
Do use a second timepicker when users need to input a range between two specific times.

Don't use when

Don't use timepicker when users need to specify a general duration or span or time. Use radio buttons or an HTML select field instead.
Use radio buttons or an HTML select field when users need to choose a general length of time.

12-hour timepicker

Use the 12-hour option for regions or users that use 12-hour time designations. This allows users to select hours, minutes, and either "AM" or "PM".

Use the 24-hour option for regions or users that use 24-hour time designations. This allows users to select hours and minutes.

Behavior

Entering time with keyboard input

Time values can be entered into the input field using the keyboard and will automatically resolve to the correct format.

States

Use 1/2 or 1/3 width columns for timepicker fields inside modals.
