The summary action bar module provides a docked container for actions and summary information on forms or multiselect lists. Within the sky-summary-action-bar component, the sky-summary-action-bar-actions component contains actions and the sky-summary-action-bar-summary component contains summary information. Within the sky-summary-action-bar-actions component, the sky-summary-action-bar-cancel component displays a cancel action, the sky-summary-action-bar-primary-action component displays a primary button, and the sky-summary-action-bar-secondary-actions component contains secondary actions that are specified with sky-summary-action-bar-secondary-action components. Within the the sky-summary-action-bar-summary component, you can specify the summary information to display. For example, you can highlight important information with key info components.

Usage

A summary action bar is used to hold actions and summary information related to the parent container to which it is docked.

When to use summary action bars
Multiselect list

Actions that can be taken on multiple items in a list of items.

When used with a multiselect list, hide the summary action bar until users select an item in the list. After users select an item, the summary action bar should fly in from the bottom of the page.
Split view

Actions that are taken on the selected record in the split view.
Modal form

You need to show summary information related to the form in a modal.
When not to use summary action bars
Large amounts of summary information

While the summary action bar can accommodate any amount of summary information, wrapping a large amount of content will eat into the vertical space on the page. Show the least amount of information that the user needs to inform their actions. Explore alternate layouts if you need to show a large amount of summary information.
Anatomy
  1. Actions
  2. Summary information (optional)
Behavior
Collapsing summary information

In smaller viewports the summary information section becomes expandable/collapsible to allow more visible space for the main content.
Options
Summary information

In cases where the user needs to see summarized information about the selected items in a list or the data entered in a form as a confirmation before taking an action, put the relevant information in the summary information area.
Conditional actions

In multiselect lists, items may be in mixed states where certain actions can only be taken on certain items. Expose all the possible actions, and append the number of items that action will be taken on when invoked.
Responsiveness
Secondary actions

Secondary actions will collapse into a context menu at small viewport widths.
Summary information

Summary information should reflow based on available width.
