Page- and modal-level help

When users perform complex tasks, they may require comprehensive documentation for an entire task instead of inline help for individual parts of the task. Users should invoke help at the page or modal level, depending on the context. In Blackbaud products, users can invoke the help service from the omnibar or a modal header bar.

Invoked inline help
Usage

Invoked inline help gives users access to help content when needed while avoiding most of the visual noise that comes with persistent inline help text. It's appropriate to use in cases where the user may need initial assistance but:

  • The task is frequently performed.
  • Ramifications of mistakes are small.

Use a help button to invoke the help content.

Depending on the complexity of the help content, you can display it in a popover or the help panel.

Layout

Place the help button to the right of the label of the element the help content supports.
Persistent inline help
Usage

Sometimes a small amount of persistent inline explanatory text can help users understand tasks. It is appropriate to use in cases where:

  • The task is complex and infrequently performed.
  • User actions have far-reaching consequences that are not obvious.

Use the "sky-deemphasized" class and limit the content to one or two sentences to minimize the visual weight of the text.

Layout

Placement of the inline help text will depend on what aspect of the task the help text supports.

  • Place overarching task help at the top of the form.
  • Place help relating to a section of the form below the section heading.
  • Place help specific to a field below the input.
