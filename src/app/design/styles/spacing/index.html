SKY UX provides two spacing approaches that can be used together when laying out content: fluid grid and spacing classes. Fluid grid works well when laying out containers and sections within containers, and the spacing classes work well when laying out content within a section.

Pages

You should use fluid grid layouts when placing containers on a page. A common example of a one-column layout is a list page – the list takes up the full width. A common example of a two-column layout is a record page – there are two columns of tiles.

 Containers

Padding will generally be handled by the SkyUX components you are using. Containers should have padding to set their content away from the container edge. The amount of padding will differ based on the size of the container.

Usage
Large containers often run near the edge of the viewport so significant padding is needed to prevent the content from feeling crunched in the viewport. Examples include:
  • Page
  • Modal
  • Tile
  • Sectioned form content area
Code
class="sky-padding-even-large"
Styling
padding: 15px
Usage
Smaller containers that are placed inside larger containers need less padding to set the content apart from its surrounding since the larger containers also have padding. Examples include:
  • Alert
  • Card
  • Popover
  • Repeater
Code
class="sky-padding-even-default"
Styling
padding: 10px
Content
Vertically-arranged content
Usage
Closely-related elements have a small amount of margin between them so that they stay visually linked. Examples include:
  • Form field labels
  • Section header and the first element in that section
Code
class="sky-margin-stacked-compact"
Styling
margin-bottom: 5px
Usage
Multiple instances of a type of object need enough vertical spacing so that the objects can be distinguished from each other, but not so much that they seem unrelated. Examples include:
  • Form inputs
Code
class="sky-margin-stacked-default"
Styling
margin-bottom: 10px
Usage
Sections of loosely-related content need extra spacing to make them easier for the user to discern. Too little spacing will make the sections visually run together.
Code
class="sky-margin-stacked-separate"
Styling
margin-bottom: 20px
Horizontally-arranged content
Usage
Closely-related elements have a small amount of margin between them so that they stay visually linked. Examples include:
  • Status indicators and text
  • Required form field indicator
  • Button icon and text
Code
class="sky-margin-inline-compact"
Styling
margin-right: 3px
Usage
Multiple instances of a type of object need enough horizontal spacing so that the objects can be distinguished from each other, but not so much that they seem unrelated. Examples include:
  • Toolbar buttons
  • Related form fields, such as "First name" and "Last name"
Code
class="sky-margin-inline-default"
Styling
margin-right: 10px
Usage
Content sections that are laid out horizontally should use a fluid grid so that they are responsive on smaller viewports. The sections should switch from a horizontal to vertical layout when there is not enough horizontal space to accommodate the width of the content.
