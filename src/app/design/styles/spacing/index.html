SKY UX provides two spacing approaches that can be used together when laying out content: fluid grid and spacing classes. Fluid grid works well when laying out containers and sections within containers, and the spacing classes work well when laying out content within sections. Pages

You should use fluid grid layouts when placing containers on a page. A common example of a one-column layout is a list page, where the list takes up the full width. A common example of a two-column layout is a record page with two columns of tiles.

 Containers

Containers require padding to set their content away from the container edge. The amount of padding differs based on the size of the container. In general, SKY UX components handle padding for you.

Usage
Large containers often run near the edge of the viewport, so significant padding is necessary to prevent the content from feeling crunched. Examples include:
  • Pages
  • Modals
  • Tiles
  • Sectioned forms
Code
class="sky-padding-even-large"
Styling
padding: 15px
Usage
Small containers within larger containers require less padding to set the content apart from its surroundings because the larger containers already have padding. Examples include:
  • Alerts
  • Cards
  • Popovers
  • Repeaters
Code
class="sky-padding-even-default"
Styling
padding: 10px
Content
Vertically arranged content
Usage
Closely related elements have a small amount of margin between them to stay visually linked. Examples include:
  • Form fields and their labels
  • Section headers and the first elements in sections
Code
class="sky-margin-stacked-compact"
Styling
margin-bottom: 5px
Usage
Multiple related elements need enough vertical spacing to be distinguished from each other but still visually linked. Examples include:
  • Form inputs
Code
class="sky-margin-stacked-default"
Styling
margin-bottom: 10px
Usage
Sections of loosely related content require extra vertical spacing to prevent them from running together visually.
Code
class="sky-margin-stacked-separate"
Styling
margin-bottom: 20px
Horizontally arranged content
Usage
Closely related elements have a small amount of margin between them to stay visually linked. Examples include:
  • Status indicator icons and labels
  • Required form field indicators
  • Button icons and labels
Code
class="sky-margin-inline-compact"
Styling
margin-right: 3px
Usage
Multiple related elements need enough horizontal spacing to be distinguished from each other but still visually linked. Examples include:
  • Toolbar buttons
  • Related form fields, such as "First name" and "Last name"
Code
class="sky-margin-inline-default"
Styling
margin-right: 10px
Usage
Use a fluid grid to lay out loosely related content sections horizontally so that they are responsive on smaller viewports. The sections should switch from a horizontal to vertical layout when there is not enough horizontal space to accommodate the width of the content.
