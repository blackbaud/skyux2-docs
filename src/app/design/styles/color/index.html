As a general rule, SKY UX uses color to convey meaning and not as pure decoration. You should rarely apply colors directly to elements because components incorporate colors as necessary. If a need for color arises outside of a component, you likely need to create a component for that scenario.

To import the color variables, add @import '~@skyux/theme/scss/mixins'; to the top of your SCSS file. Do not include node_modules in the path because the SKY UX package may not always be in the root of node_modules and ~ resolves the dependency in those cases.

After you import the colors, you can reference them by the variable names that are listed in the Code columns below.

Text

Ensure that the contrast between the text color and the background color is at least 4.5:1 to meet Web Content Accessibility Guidelines 2.0 level AA.

Modified states

In general, hover states for active elements apply a 10 percent darken to the element's background, border, and/or text.

In general, disabled states for active elements apply .65 opacity to the element.

Within alerts and toasts, hyperlinks are automatically styled to maintain accessibility.
