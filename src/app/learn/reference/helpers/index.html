This reference describes the helpers that are available to make authenticated HTTP requests.

External developers should keep in mind that some SKY UX options in the technical references are for internal Blackbaud use only. External developers can use SKY UX to create, test, and run SKY UX SPAs, and the SKY UX components, services, and modules are open source and available in multiple repositories that target different areas of development. However, not all SKY UX features are available for external use.

  • import { SkyHostBrowser } from '@skyux-sdk/e2e — Particularly useful when running the skyux e2e command in the SKY UX CLI. Use SkyHostBrowser.get() to automatically add the SKY UX Host base URL, as well as the necessary configuration object to enable local development mode. For example, SkyHostBrowser.get(/my-page?my-qs=1');.

  • import { SkyAuthHttpClientModule } from '@skyux/http'; — Enables SKY UX SPAs to use the HttpClient Angular service instead of the deprecated Http service. Use the skyAuthHttpOptions() function in HTTP calls to capture Blackbaud URL params such as leid and envid. This helper is for internal Blackbaud use only. For Blackbaud developers, the Engineering System website describes how to use SkyAuthHttpClientModule to make authenticated calls from a SPA to a web service.
