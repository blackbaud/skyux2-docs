skyux pact runs pact tests. Pact is a testing framework that you can use to test a SPA's integration with the services that it consumes. Karma is configured to run any files in the src/app folder with names that end in .pact-spec.ts. The pact command reads the pacts setting in the skyuxconfig.json configuration file and spins up an individual pact server for each provider service specified in the pacts property. It also spins up a pact server for the proxy server. We recommend creating pact tests for all services that the SPA consumes to confirm that your SPA can call out to the provider services and vice versa. For information about how to configure the SPA for pact testing, see the configuration reference. And for more information on configuration settings and guidance on how to write pact tests, see the pact testing tutorial.
