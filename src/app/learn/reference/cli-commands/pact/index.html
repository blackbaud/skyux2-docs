skyux pact runs pact tests. Pact is a testing framework that you can use to test a SPA's integration with the services that it consumes. We recommend creating pact tests for all services that the SPA consumes to confirm that your SPA can call out to the provider services and vice versa. Karma is configured to run any files in the src/app folder with names that end in .pact-spec.ts. The pact command reads the pacts setting in the skyuxconfig.json configuration file and spins up an individual pact server for each provider service specified in the pacts property.

The pact command also spins up a pact server for the proxy server. It acts as a handler for all pact server requests from the browser and runs them in the host and port specified in your pacts configuration settings. To fulfil CORS when requests are made to the pact server, the proxy server changes the Origin headers to the url value that the host property defines in skyuxconfig.json. This url specifies a base URL to pass information to SKY UX Host, and its default value is https://host.nxt.blackbaud.com. This ensures that pact tests accurately represent interactions between consumers and providers.

For information about how to configure the SPA for pact testing, see the configuration reference. And for more information on configuration settings and guidance on how to write pact tests, see the pact testing tutorial.
