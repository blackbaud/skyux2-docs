To gracefully handle requests to unknown routes within a SPA, SKY UX Builder creates a default wildcard route known as the 404 page. The component on the 404 page displays the "Errors SPA" in an iframe, ultimately using the error component with an errorType of notfound.

To customize the wildcard route, you can create a not-found.component.ts file in src/app, import the Component class from Angular to provide functionality to the component, define the template for the component, and export the NotFoundComponent component class:

import { Component } from '@angular/core'; @Component({ templateUrl: './not-found.component.html' }) export class NotFoundComponent {}

You can then create a not-found.component.html file in src/app and provide the content for the wildcard route to display:

This is my custom 404 page.

Please note that while the filename is technically not important to this functionality, the component name is crucial. The component must export NotFoundComponent.
