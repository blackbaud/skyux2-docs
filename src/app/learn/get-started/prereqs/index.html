To set up your developer machine for SKY UX, make sure you have the following prerequisites in place.

Node.js

SKY UX highly recommends the latest LTS Dubnium version of Node.js. To check your version, run node –v from the command line.

In addition, SKY UX recommends Node Version Manager (NVM) to wrap your Node.js installation. NVM installs Node.js in your user directory to avoid permissions-related issues, and it also allows you to easily upgrade and manage multiple Node.js versions.

  • For Mac OS X and Linux, follow the installation and update instructions in the README.md file for the NVM GitHub repo.
  • For Windows, follow the installation and update instructions in the README.md file for the NVM for Windows GitHub repo.

For Windows machines, you must also install required tools and configurations using Microsoft's windows-build-tools. Run npm install --global --production windows-build-tools as an administrator from an elevated PowerShell or CMD.exe.

NPM

We require at least version 3 of NPM. NPM is the default JavaScript package manager for Node.js. As of Node.js version 0.6.3, it is bundled and installed automatically with the environment. To check your version, run npm –v from the command line. To install a specific version, run npm install globally and specify the version. For example: npm install -g npm@4.

Git

Install Git. We do not require a specific version but recommend you install the latest version. To confirm you have Git, run git version from the command line.

SKY UX CLI

The SKY UX CLI provides a command line interface for the SKY UX Builder that builds the output for SKY UX applications. To install the CLI from the command line, run npm install -g @skyux-sdk/cli.

SKY UX certificate

For local development with SKY UX, you must install the self-signed certificate that enables your SPA to securely load local files. To add the certificate to your machine's list of trusted certificates and validate it, run the following command:

skyux certs install

The certificate is required to securely run skyux serve, skyux e2e, and skyux build --serve. For more information, see the skyux certs documentation.

Additional steps for Firefox

  1. Run skyux serve to serve your SKY UX SPA.

  2. Enter "about:preferences#privacy" in the Firefox address bar.

  3. Near the bottom of the page, select View Certificates.

  4. Under Servers on the Certificate Manager, select Add Exception.

  5. Specify the location to use the certificate. The default location is https://localhost:8000, but you can configure a different port in the skyuxconfig.json file or set up command-specific ports in additional configuration files.

    To serve multiple SPAs, you must also specify the next available port. When you serve multiple SPAs, SKY UX uses the next available port after the first port is in use.

  6. Select Get Certificate, and then select Confirm Security Exception.

Additional steps for iOS Simulator on Mac

  1. Open the .skyux/certs folder in your home directory.

  2. Drag and drop the SKY UX certificate to your simulator.

  3. In the prompt that appears, select Allow to download the profile and then select Close.

  4. Open the Settings app and navigate to General > Profile > SKY UX.

  5. Select Install on each page (three times) to install the certificate.

  6. Select Done to exit.

  7. In Settings > General > About, select Certificate Trust Settings.

  8. Select Enable Full Trust For Root Certificates for the SKY UX certificate and continue when prompted.
