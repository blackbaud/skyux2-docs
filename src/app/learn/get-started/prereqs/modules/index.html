SKY UX modules provide Angular components, directives, and services to enable developers that create user interfaces that incorporate SKY UX styles and patterns. The framework provides functionality through NPM packages targeting different areas of development, and developers must intall the modules in those packages as dependencies in their projects. To learn how SKY UX organizes packages, see the packages matrix.

Use modules in SKY UX SPAs

To use exports from a SKY UX module, you must install its package as a dependency to your project.

  1. In the command prompt, run the following command from your SPA's root directory. Replace [package-name] with the name of the package that contains the module.

    npm install [package-name] --save --save-exact

    For example, the @skyux/modals package contains the SkyConfirmModule and SkyModalModule modules. To install it, run the following command:

    npm install @skyux/modals --save --save-exact

  2. After you install a package, check the command prompt for missing dependencies and install them as necessary.

  3. To make exports from a module available to your SPA, add the module to the exports section of your SPA's app-sky.module.ts file.
