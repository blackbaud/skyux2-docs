After you serve the default single-page application, you can edit the existing pages and components and add additional pages and components as necessary.

Edit an existing page

  1. With skyux serve running, navigate to the default site's About page in your web browser.

  2. In a local editor, open the project's about.component.html file in the src/app/about folder.

    About our Team

    Warning! We never check our email.
    {{ member.name }} {{ member.email }}

  3. At the top of the file before the warning, add a paragraph.

    This page provides contact information for members of our team.

  4. Save the changes. The skyux serve command refreshes the About page in the web browser to display the new paragraph.

Edit a component on an existing page

  1. With skyux serve running, return to the project’s about.component.html file in a local editor.

  2. Edit the type and message for the alert component. For example, in the alertType attribute, change the value to from warning to info. And in the sky-alert element, change the message to There may be a delay in responding to your email.

    Note: There may be a delay in responding to your email.

  3. Save the changes. The skyux serve command refreshes the About page in the web browser to display the updated background color and text.

Add a component to an existing page

  1. With skyux serve running, return to the project’s about.component.html file in a local editor.

  2. Between the alert and avatar components, add a card component. Add the sky-card element, and then within that element, add the sky-card-title element for the header text, the sky-card-content element for the body content, and the sky-card-actions element for the button.

    Large card This card demonstrates the amount of space that is available for a card that uses the default large size. If the content does not require this much space, you can set the card size to small. The type of content to display in the body of a card varies based on what the card represents and whether it prompts users to action. The content can consist of some combination of a graph, trend, status, summary information, and guidance text.

  3. Save the changes. The skyux serve command refreshes the About page in the web browser to display the new card component between the alert and avatar components.

Create a page

By default, SKY UX uses automatic route configuration based on the project's folder structure. So to create a page, you simply create a folder in the existing src/app folder and then create an index.html file in that folder.

  1. In a local editor, create a folder within src/app and name it demo. The folder name appears in the route for the page.

  2. In the demo folder, create an index.html file.

  3. In the new file, place the shared app-nav component at the top and then add hello world content such as <h1>Hello world!</h1>. The app-nav component is an example of a shared component. It manually creates top-level navigation within your app.

    Hello World!

  4. In the src/assets/locales folder, open the resources_en_US.json file and add a localization string for a top-level navigation item to access the Demo page.

    "app_nav_demo": { "message": "Demo", "_description": "The text for the Demo top-level navigation item" },

  5. In the src/app/shared folder, open the app-nav.component.ts file and add a path for the Demo page. Set titleKey to the localization string we just added to resources_en_US.json, and set path to "/demo." titleKey specifies a label for the menu, and path specifies the location of the folder.

    import { Component } from '@angular/core'; @Component({ selector: 'app-nav', templateUrl: './app-nav.component.html', styleUrls: ['./app-nav.component.scss'] }) export class AppNavComponent { public nav = [ { titleKey: 'app_nav_home', path: '/' }, { titleKey: 'app_nav_about', path: '/about' }, { titleKey: 'app_nav_demo', path: '/demo' } ]; }

  6. Save the changes. If skyux serve is running, it refreshes the SPA to display the Demo menu item. If not, enter skyux serve in the command prompt to launch the SPA.

  7. Click Demo in the menu. The new page appears.

Create a component

After you add basic content to your page, you may need more programmatic control over the contents of a page. For example, you might want to display dynamic data based on a web request or client-side logic written in TypeScript. To do that, you need to create your own component.

The SKY UX CLI makes it easy to create components from the command line. For this example, we'll create a component in the demo folder from the previous tutorial.

  1. From the command line, run skyux generate component demo/my-demo in the context of your project's root directory. This creates the TypeScript, HTML, SCSS, and spec files to build and test your component.

  2. In your code editor, navigate to the demo folder, and open the my-demo.component.ts file that includes boilerplate code to create your component.

    import { Component } from '@angular/core'; @Component({ selector: 'app-my-demo', templateUrl: './my-demo.component.html', styleUrls: ['./my-demo.component.scss'] }) export class MyDemoComponent { }

  3. We want our component to accept input from the host page. This requires a public property with the Angular @Input decorator. We can import this decorator from @angular/core. For this example, we'll add an input property called message.

    import { Component, Input } from '@angular/core'; @Component({ selector: 'app-my-demo', templateUrl: './my-demo.component.html', styleUrls: ['./my-demo.component.scss'] }) export class MyDemoComponent { @Input() public message: string; }

  4. Now that the component accepts input, we can display input in the component's template. Open the my-demo.component.html file, and add the following code to display the message.

    {{ message }}

    The curly braces are the syntax to reference public properties of the component class.

  5. Open the index.html file in the demo folder, and add the new app-my-demo component after the header. Set the value of the message attribute to "Hello!"

  6. Save the changes, then return to the command prompt and run skyux serve to launch the SPA in your default browser. On the SPA's Demo page, the "Hello!" message is rendered after the header.
