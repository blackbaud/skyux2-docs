After you install the prerequisites for SKY UX, you can use the SKY UX CLI to create a local single-page application. Before you create the SPA, we recommend that you set up a Git repo for its source files. That way, the CLI can clone the repo and automatically pull in the SKY UX template. However, to work locally only, you can create the SPA without a repo.

Blackbaud developers should create repos and provision applications through the SKY UX App Management Portal. Documentation on the SKY UX App Management Portal is available on the Engineering System website. This site is for Blackbaud internal use only and is not available to external developers.

  1. From the command prompt, cd into the directory where you want to create the application.

    Do not use Git Bash with Windows because known issues occur with skyux new. We recommend the native Windows command prompt instead.

  2. Enter skyux new to initialize a new local SKY UX project. This pulls in the latest version of the template and runs npm install to install the necessary dependencies.

    By default, skyux new uses the SKY UX application template that includes the necessary items for a SKY UX single-page application. If necessary, you can specify a different template such as the SKY UX Add-in template or the Stache template. For more information about templates, see the SKY UX CLI reference.

    After you enter the command, a series of prompts appears.

  3. In the first prompt, specify a root directory for the application. The SKY UX CLI creates a folder with this name and prefixes it with "skyux-spa." The root directory determines how to access the SPA because the URL for the home page includes the root directory after the base URL.

  4. In the second prompt, specify the URL for a Git repo to store the source files for the application. You can use a Visual Studio Team System Git repo or a GitHub repo. The SKY UX CLI clones the repo and automatically pulls in the SKY UX template.

    To work locally only, you can skip the second prompt and create the SPA without a repo. Later, you can manually add a repo as a remote to the local application. To do this, enter git remote add origin [URL] in the command prompt, where "[URL]" is the path for the repo. Then to push local content to the external repo, enter git push origin --all.

  5. After the SKY UX CLI downloads the SKY UX template project and runs npm install to install dependencies, the command prompt displays the directory name: "SPA [directory name] created in directory "skyux-spa-[directory name]."
