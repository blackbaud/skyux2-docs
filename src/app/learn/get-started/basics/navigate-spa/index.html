SKY UX takes advantage of the Angular Router class to enable navigation within a SPA, and you can use Angular's RouterLink directive to create links between pages in a SPA. In addition, SKY UX's SkyAppLink and SkyAppLinkExternal directives extend the RouterLink directive and allow you to create links that automatically include parameters in the URL. Create internal links within a SPA

To create links between pages in a SPA, use the HTML a element.

  • If the SPA's URL does not include parameters, you can use the routerLink attribute to specify the path to link to.

    Sample Link 1

    This sample directs users to /sample-spa/sample-route and does not include any parameters in the URL.

  • If the SPA's URL includes parameters, you can use the skyAppLink attribute to specify the path to link to. The SkyAppLink directive automatically includes the SPA's parameters in the URL. The most common parameters are envid and svcid, but the URL can include any parameters that are provided to the SPA when it loads.

    Sample Link 2

    If the SPA has a svcid of "abc" and an envid of "1234," this sample directs users to /sample-spa/sample-route?svid=abc&envid=1234.

    You can also use routerLink to specify the path to link to and then use the queryParams attribute to manually specify the SPA's parameters, but we recommend letting the SkyAppLink directive handle this step for you.

Create external links to another SPA

To create links to other SPAs, use the HTML a element.

  • If the external SPA's URL does not include parameters, you can use the standard HTML href attribute to specify the URL to link to.

    Sample Link 3

    This sample leaves the current SPA and directs users to https://www.external-spa.com/sample-path. It does not include any parameters in the URL.

  • If the SPA's URL includes parameters, you can use the skyAppLinkExternal attribute to specify the path to link to. The SkyAppLinkExternal directive automatically includes the external SPA's parameters in the URL. The most common parameters are envid and svcid, but the URL can include any parameters that are provided to the SPA when it loads.

    Sample Link 4

    If the external SPA has a svcid of "abc" and an envid of "1234," this sample directs users to /another-spa/sample-route?svid=abc&envid=1234.
