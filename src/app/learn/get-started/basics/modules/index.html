SKY UX modules provide Angular components, directives, pipes, and services. The framework provides functionality through NPM packages that target different areas of development, and developers install the modules in those packages as dependencies in their projects. To learn how SKY UX organizes packages, see the packages matrix.

Install SKY UX modules

To use exports from a SKY UX module, you must install its package as a dependency to your project.

  1. In the command prompt, run the following command from your SPA's root directory, where [package-name] is the name of the package that contains the module.

    npm install --save --save-exact [package-name]

  2. While the install command runs, watch the command prompt for warnings about missing peerDependencies, and then install those dependencies as necessary.

  3. Make the exports from the module available to your SPA.

    • For SKY UX SPAs, add the module to the exports section of the SPA's app-sky.module.ts file.
    • For non-SKY UX SPAs, add the module to the exports section of the SPA's main module, which is usually the app.module.ts file.
    • For libraries, add the module to the imports section of the library module.
Update SKY UX modules

To update the packages and modules that you install as dependencies in your project, run skyux upgrade.

The skyux upgrade command currently requires you to install @skyux-sdk/builder-plugin-migrate globally. We are exploring moving this command to the main SKY UX CLI package. For more information, see the SKY UX migration guide.

This command pulls in the latest version of the devDependencies and peerDependencies as well as any dependencies listed in the SPA's package.json.
