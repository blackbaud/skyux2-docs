Before you build and publish your SPA, you must choose a web server to host your site. We recommend GitHub Pages, a free tool suitable for front-end projects.

Blackbaud developers create repos and provision applications through the SKY UX App Management Portal. Documentation on the SKY UX App Management Portal is available on the Engineering System website. This site is for Blackbaud internal use only and is not available to external developers.

  1. From the command prompt, cd into the application's directory.

  2. Run skyux build to build the SPA into the dist/ folder.

  3. Copy the contents of the dist/ folder into a directory with the SPA name that you specified when you created the SPA. For example, if you named the SPA my-spa when you ran skyux new, change the dist/ folder to my-spa/.

    To host your SPA at the root without a subfolder, set the base property to / in the app configuration section in skyuxconfig.json.

  4. Host the my-spa/ directory on your preferred web server.

  5. In a browser, enter https://example.com/my-spa to view your SPA.

    SKY UX uses HTML5 push states for routing, so you must either configure your host to redirect all requests to the root index.html or enable the useHashRouting configuration option.
