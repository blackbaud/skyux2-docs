Before you build and publish your SPA, you must first choose a web server to host your site. We recommend GitHub Pages, a free tool suitable for front-end projects.

Blackbaud developers should create repos and provision applications through the SKY UX App Management Portal. Documentation on the SKY UX App Management Portal is available on the Engineering System website. This site is for Blackbaud internal use only and is not available to external developers.

  1. From the command prompt, cd into the application's directory.

  2. Enter skyux build to build the SPA into the dist/ folder.

  3. Copy the contents of the dist/ folder into a directory using the SPA name specified in create a SPA.

    For example, if you provided my-spa after running skyux new, change the dist/ folder to my-spa/.

  4. Host the my-spa/ directory on your preferred web server.

  5. In a browser, enter https://example.com/my-spa to view your SPA.

    SKY UX uses HTML5 push states for routing, so you must either configure your host to redirect all requests to the root index.html or enable the useHashRouting configuration option.
